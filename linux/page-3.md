## 1. PENDAHULUAN & SEJARAH LINUX

### 1.1 Apa itu Linux?

Linux adalah sistem operasi berbasis **kernel open-source** yang dikembangkan pertama kali oleh **Linus Benedict Torvalds** pada tahun 1991. Linux bukan hanya sekadar sistem operasi — ia adalah fondasi dari ribuan distribusi yang digunakan di server, desktop, smartphone (Android), superkomputer, embedded systems, dan cloud computing.

Linux mengikuti filosofi **UNIX** — sistem operasi multi-user, multi-tasking yang dikembangkan oleh Bell Labs pada akhir 1960-an. Berbeda dengan sistem operasi proprietary seperti Windows atau macOS, Linux didistribusikan secara bebas dan terbuka di bawah lisensi **GNU General Public License (GPL)**.

### 1.3 Filosofi Linux

#### Filosofi UNIX (The Unix Philosophy)

1. **Tulis program yang melakukan satu hal dan melakukannya dengan baik**
2. **Tulis program yang bekerja bersama**
3. **Tulis program yang menangani text stream, karena itu adalah antarmuka universal**

#### Prinsip-Prinsip Utama Linux

* **Open Source** — Kode sumber tersedia untuk umum
* **Free Software** — Bebas digunakan, dipelajari, dimodifikasi, didistribusikan
* **Community-driven** — Dikembangkan oleh komunitas global
* **Stability** — Dirancang untuk uptime tinggi
* **Security** — Model keamanan berbasis permission yang kuat
* **Portability** — Berjalan di berbagai arsitektur hardware

### 2.2 Kernel Linux

Kernel adalah inti dari sistem operasi Linux. Ia bertanggung jawab untuk:

#### Fungsi Utama Kernel

```
text1. MANAJEMEN PROSES   ├── Penjadwalan proses (scheduling)   ├── Pembuatan dan penghapusan proses   ├── Komunikasi antar proses (IPC)   └── Thread management2. MANAJEMEN MEMORI   ├── Virtual memory management   ├── Paging dan swapping   ├── Memory allocation   └── Cache management3. MANAJEMEN FILE SYSTEM   ├── Virtual File System (VFS)   ├── Buffer cache   ├── File dan direktori operations   └── Dukungan berbagai format filesystem4. MANAJEMEN PERANGKAT   ├── Device drivers   ├── Interrupt handling   ├── DMA (Direct Memory Access)   └── Hardware abstraction5. MANAJEMEN JARINGAN   ├── Network protocols (TCP/IP)   ├── Socket interface   ├── Packet filtering (iptables/nftables)   └── Network device drivers
```

#### Jenis Kernel

| Jenis           | Deskripsi                     | Contoh                  |
| --------------- | ----------------------------- | ----------------------- |
| **Monolithic**  | Semua layanan di kernel space | Linux, Unix tradisional |
| **Microkernel** | Layanan minimal di kernel     | MINIX, QNX              |
| **Hybrid**      | Kombinasi keduanya            | Windows NT, macOS XNU   |
| **Exokernel**   | Minimal abstraction           | MIT Exokernel           |

> Linux menggunakan **Monolithic kernel** dengan **Loadable Kernel Modules (LKM)** yang memungkinkan modularitas.

#### Jenis Partition Table

```
textMBR (Master Boot Record):  - Maksimal 4 primary partitions  - Maksimal disk size 2 TB  - Kompatibel dengan BIOS lama  - Boot sector di sektor pertama diskGPT (GUID Partition Table):  - Hingga 128 partitions  - Mendukung disk > 2 TB  - Memerlukan UEFI (atau BIOS dengan CSM)  - Lebih redundan dan aman

```
text/├── bin/          → Essential user binaries├── boot/         → Boot loader files├── dev/          → Device files├── etc/          → System configuration files├── home/         → User home directories│   ├── user1/│   └── user2/├── lib/          → Essential shared libraries├── lib64/        → 64-bit libraries├── media/        → Mount point for removable media├── mnt/          → Temporary mount points├── opt/          → Optional application software├── proc/         → Virtual filesystem (process info)├── root/         → Root user home directory├── run/          → Runtime data├── sbin/         → Essential system binaries├── srv/          → Service data├── sys/          → Virtual filesystem (kernel/hardware)├── tmp/          → Temporary files├── usr/          → Secondary hierarchy│   ├── bin/      → User commands│   ├── include/  → Header files│   ├── lib/      → Libraries│   ├── local/    → Locally installed software│   ├── sbin/     → System commands│   └── share/    → Architecture-independent data└── var/          → Variable data    ├── log/      → Log files    ├── cache/    → Application cache    ├── mail/     → User mailboxes    ├── spool/    → Spool data    └── www/      → Web server files

### 5.2 Penjelasan Detail Setiap Direktori

#### / (Root)

```
textDirektori paling atas dalam hierarki filesystem.Semua file dan direktori dimulai dari root.Hanya root user yang dapat memodifikasi konten langsung di /.
```

#### /bin

```
textBinary executables yang diperlukan oleh semua pengguna.Tersedia bahkan ketika /usr tidak di-mount.Contoh isi:  bash, cat, cp, date, dd, df, echo, grep,  gzip, hostname, kill, ln, ls, mkdir, more,  mount, mv, rm, sed, sh, tar, touch, vi
```

#### /boot

```
textFile yang diperlukan untuk proses booting.Isi:  vmlinuz          → Kernel Linux yang dikompres  initrd.img       → Initial RAM disk  grub/            → GRUB bootloader files  System.map       → Kernel symbol map  config-*         → Kernel build configuration
```

#### /dev

```
textDevice files — representasi hardware sebagai file.Jenis device files:  Block devices  → /dev/sda, /dev/nvme0n1 (storage)  Character devices → /dev/tty, /dev/random  Pseudo devices → /dev/null, /dev/zero, /dev/urandomContoh:  /dev/sda      → First SATA hard disk  /dev/sda1     → First partition of first SATA disk  /dev/nvme0n1  → First NVMe disk  /dev/tty1     → First virtual terminal  /dev/null     → Discard everything written to it  /dev/zero     → Provides infinite zero bytes  /dev/random   → Random number generator  /dev/urandom  → Non-blocking random number generator  /dev/stdin    → Standard input  /dev/stdout   → Standard output  /dev/stderr   → Standard error
```

#### /etc

```

#### /home

```
textHome direktori untuk setiap pengguna.Struktur:  /home/username/    ├── .bashrc       → Bash configuration    ├── .bash_profile → Login shell config    ├── .ssh/         → SSH keys & config    ├── .config/      → Application configs    ├── Desktop/    ├── Documents/    ├── Downloads/    ├── Music/    ├── Pictures/    └── Videos/
```

#### /proc

```
textVirtual filesystem yang berisi informasi tentang proses dan kernel.File di /proc tidak ada secara fisik di disk.Contoh:  /proc/cpuinfo       → CPU information  /proc/meminfo       → Memory usage  /proc/version       → Kernel version  /proc/uptime        → System uptime  /proc/loadavg       → Load average  /proc/mounts        → Mounted filesystems  /proc/net/          → Network statistics  /proc/sys/          → Kernel parameters (sysctl)  /proc/[PID]/        → Per-process information    ├── cmdline       → Command yang menjalankan proses    ├── status        → Status proses    ├── fd/           → File descriptors    └── maps          → Memory maps

#### /sys

```
textVirtual filesystem yang menyediakan interface ke kernel data structures.Contoh:  /sys/class/net/     → Network interfaces  /sys/block/         → Block devices  /sys/bus/           → Bus subsystems  /sys/devices/       → Device tree  /sys/kernel/        → Kernel parameters  /sys/power/         → Power management

#### /tmp

```
textTemporary files.Biasanya dibersihkan saat reboot atau secara berkala.Semua pengguna dapat menulis ke /tmp (dengan sticky bit).Karakteristik:  - Sticky bit aktif (chmod +t)  - Ukuran terbatas  - Sering di-mount sebagai tmpfs (RAM)
