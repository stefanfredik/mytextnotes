## 1. PENDAHULUAN & SEJARAH LINUX

### 1.1 Apa itu Linux?

Linux adalah sistem operasi berbasis **kernel open-source** yang dikembangkan pertama kali oleh **Linus Benedict Torvalds** pada tahun 1991. Linux bukan hanya sekadar sistem operasi — ia adalah fondasi dari ribuan distribusi yang digunakan di server, desktop, smartphone (Android), superkomputer, embedded systems, dan cloud computing.

Linux mengikuti filosofi **UNIX** — sistem operasi multi-user, multi-tasking yang dikembangkan oleh Bell Labs pada akhir 1960-an. Berbeda dengan sistem operasi proprietary seperti Windows atau macOS, Linux didistribusikan secara bebas dan terbuka di bawah lisensi **GNU General Public License (GPL)**.

### 1.3 Filosofi Linux

#### Filosofi UNIX (The Unix Philosophy)

1. **Tulis program yang melakukan satu hal dan melakukannya dengan baik**
2. **Tulis program yang bekerja bersama**
3. **Tulis program yang menangani text stream, karena itu adalah antarmuka universal**

#### Prinsip-Prinsip Utama Linux

* **Open Source** — Kode sumber tersedia untuk umum
* **Free Software** — Bebas digunakan, dipelajari, dimodifikasi, didistribusikan
* **Community-driven** — Dikembangkan oleh komunitas global
* **Stability** — Dirancang untuk uptime tinggi
* **Security** — Model keamanan berbasis permission yang kuat
* **Portability** — Berjalan di berbagai arsitektur hardware

### 2.2 Kernel Linux

Kernel adalah inti dari sistem operasi Linux. Ia bertanggung jawab untuk:

#### Fungsi Utama Kernel

{% code overflow="wrap" %}
```
1. MANAJEMEN PROSES   
├── Penjadwalan proses (scheduling)  
├── Pembuatan dan penghapusan proses   
├── Komunikasi antar proses (IPC)   
    └── Thread management2. MANAJEMEN MEMORI   
├── Virtual memory management   
├── Paging dan swapping   
├── Memory allocation   
    └── Cache management3. MANAJEMEN FILE SYSTEM   
├── Virtual File System (VFS)   
├── Buffer cache   
├── File dan direktori operations   
    └── Dukungan berbagai format filesystem4. MANAJEMEN PERANGKAT   
├── Device drivers   
├── Interrupt handling   
├── DMA (Direct Memory Access)   
    └── Hardware abstraction5. MANAJEMEN JARINGAN   
├── Network protocols (TCP/IP)   
├── Socket interface   
├── Packet filtering (iptables/nftables)   
    └── Network device drivers
```
{% endcode %}

#### Jenis Kernel

| Jenis           | Deskripsi                     | Contoh                  |
| --------------- | ----------------------------- | ----------------------- |
| **Monolithic**  | Semua layanan di kernel space | Linux, Unix tradisional |
| **Microkernel** | Layanan minimal di kernel     | MINIX, QNX              |
| **Hybrid**      | Kombinasi keduanya            | Windows NT, macOS XNU   |
| **Exokernel**   | Minimal abstraction           | MIT Exokernel           |

> Linux menggunakan **Monolithic kernel** dengan **Loadable Kernel Modules (LKM)** yang memungkinkan modularitas.

## 4. INSTALASI LINUX

### 4.1 Persiapan Instalasi

#### Kebutuhan Sistem Minimum (Ubuntu 22.04 LTS)

```
Processor  : 2 GHz dual-core atau lebih baik
RAM        : 4 GB (minimum 2 GB)
Storage    : 25 GB ruang disk
Display    : 1024 x 768 resolusi
Internet   : Direkomendasikan untuk download update
USB/DVD    : Untuk media instalasi
```

#### Langkah-Langkah Persiapan

```
1. Download ISO image dari website resmi
2. Verifikasi checksum file ISO
3. Buat bootable USB/DVD
4. Backup data penting
5. Periksa kompatibilitas hardware
6. Siapkan partisi (jika dual boot)
```

### 4.2 Verifikasi ISO

```bash
# Verifikasi SHA256 checksum (Ubuntu)
# Download SHA256SUMS dari website Ubuntu

# Di Linux/Mac
sha256sum ubuntu-22.04.3-desktop-amd64.iso

# Bandingkan dengan checksum resmi
cat SHA256SUMS | grep ubuntu-22.04.3-desktop-amd64.iso

# Verifikasi GPG signature
gpg --keyid-format long --verify SHA256SUMS.gpg SHA256SUMS
```

### 4.3 Membuat Bootable USB

#### Menggunakan dd (Linux/Mac)

```bash
# Identifikasi USB drive
lsblk
# atau
fdisk -l

# Buat bootable USB (HATI-HATI: pastikan /dev/sdX benar!)
sudo dd if=/path/to/ubuntu.iso of=/dev/sdX bs=4M status=progress sync

# Contoh:
sudo dd if=ubuntu-22.04.3-desktop-amd64.iso of=/dev/sdb bs=4M status=progress sync
```

#### Menggunakan Rufus (Windows)

```
1. Download Rufus dari rufus.ie
2. Pilih USB drive
3. Pilih file ISO
4. Pilih partition scheme:
   - GPT untuk UEFI
   - MBR untuk Legacy BIOS
5. Klik START
6. Tunggu hingga selesai
```

#### Menggunakan Balena Etcher (Cross-platform)

```
1. Download dari balena.io/etcher
2. Pilih Flash from file → pilih ISO
3. Pilih target → pilih USB drive
4. Klik Flash!
```

### 4.4 Partisi Disk

#### Skema Partisi Rekomendasi

**Untuk Sistem Single Boot (Desktop)**

```

swap         : 8-16 GB (swap)
```

**Untuk Dual Boot (Linux + Windows)**

```

[Linux Partisi]
  /           : 30 GB  (ext4)
  /home       : Sisa   (ext4)
  swap        : 4 GB   (swap)
```

#### Jenis Partition Table

```
MBR (Master Boot Record):
  - Maksimal 4 primary partitions
  - Maksimal disk size 2 TB
  - Kompatibel dengan BIOS lama
  - Boot sector di sektor pertama disk

GPT (GUID Partition Table):
  - Hingga 128 partitions
  - Mendukung disk > 2 TB
  - Memerlukan UEFI (atau BIOS dengan CSM)
  - Lebih redundan dan aman
```

### 5.2 Penjelasan Detail Setiap Direktori

#### / (Root)

```
Direktori paling atas dalam hierarki filesystem.
Semua file dan direktori dimulai dari root.
```

#### /bin

```
Direktori paling atas dalam hierarki filesystem.
Semua file dan direktori dimulai dari root.
Hanya root user yang dapat memodifikasi konten langsung di /.
```

#### /boot

```
File yang diperlukan untuk proses booting.

Isi:
```

#### /dev

```
Device files — representasi hardware sebagai file.

Jenis device files:
  Block devices  → /dev/sda, /dev/nvme0n1 (storage)
  Character devices → /dev/tty, /dev/random
  Pseudo devices → /dev/null, /dev/zero, /dev/urandom
```

```
  /proc/mounts        → Mounted filesystems
```

```bash
#   - Bisa untuk file dan direktori
```

### 8.1 Konsep User dan Group

```
Linux menggunakan sistem User/Group untuk:
- Mengidentifikasi pengguna
- Mengontrol akses ke file dan resources
- Memisahkan privilege

Tipe User:
  Root (UID=0)     → Superuser, akses penuh
  System Users     → UID 1-999, untuk daemon/services
  Regular Users    → UID >= 1000, pengguna biasa

File terkait:
  /etc/passwd      → Database user
  /etc/shadow      → Password terenkripsi
  /etc/group       → Database group
  /etc/gshadow     → Password group terenkripsi
```

### 8.2 File /etc/passwd

```
Format: username:password:UID:GID:GECOS:home:shell

root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
ubuntu:x:1000:1000:Ubuntu,,,:/home/ubuntu:/bin/bash

Field penjelasan:
  username  → Nama pengguna (max 32 karakter)
  password  → 'x' berarti di /etc/shadow
  UID       → User ID (angka unik)

  GECOS     → Informasi tambahan (nama, nomor, dll)
  home      → Home direktori
  shell     → Login shell default
```

### 8.3 File /etc/shadow

```
Format: username:password:lastchange:min:max:warn:inactive:expire

ubuntu:$6$rounds=5000$salt$hash:19001:0:99999:7:::

Field penjelasan:
  username    → Nama pengguna
  password    → Password terenkripsi ($6$ = SHA-512)
  lastchange  → Hari sejak Unix Epoch (Jan 1, 1970) terakhir ganti
  min         → Minimum hari sebelum bisa ganti password
  max         → Maximum hari sebelum harus ganti password
  warn        → Hari peringatan sebelum expire
  inactive    → Hari setelah expire sebelum akun disabled
  expire      → Tanggal akun kedaluwarsa (kosong = tidak pernah)
```

## 9. PERMISSION & OWNERSHIP

### 9.1 Konsep Permission Linux

```
Linux menggunakan sistem permission 3-tier:
  Owner (User)  → Pemilik file
  Group         → Grup yang memiliki file
  Others        → Semua pengguna lain

Tipe Permission:
  r (read)      → Membaca file / list direktori  = 4
  w (write)     → Menulis ke file / modify dir   = 2
  x (execute)   → Eksekusi file / masuk direktori = 1

Format ls -l:
  -rwxr-xr--  1  user  group  4096  Jan 15  file.txt
  │└─┬──┘└─┬─┘
  │  │     └── Others: r-- (4)
  │  │     Permission
  │  └────── Group: r-x (5)
  │  Permission
  └─────── Owner: rwx (7)
  Permission
  │
  └── File type:
      - Regular file
      d Directory
      l Symbolic link
      b Block device
      c Character device
      p Named pipe (FIFO)
      s Socket
```

### 9.2 Membaca Permission

```
-rwxr-xr--

Posisi: 1  234  567  890
         │  │    │    │
         │  │    │    └── Others
         │  │    └─────── Group
         │  └──────────── Owner
         └─────────────── File type

Baca:
  Owner : rwx = 4+2+1 = 7 (baca, tulis, eksekusi)
  Group : r-x = 4+0+1 = 5 (baca, eksekusi)
  Others: r-- = 4+0+0 = 4 (hanya baca)

Permission Value:
  0 = ---  (no permission)
  1 = --x  (execute only)
  2 = -w-  (write only)
  3 = -wx  (write + execute)
  4 = r--  (read only)
  5 = r-x  (read + execute)
  6 = rw-  (read + write)
  7 = rwx  (read + write + execute)
```

### 9.3 Mengubah Permission (chmod)

```bash
# MODE NUMERIK (Octal)
chmod 755 file.txt       # rwxr-xr-x
chmod 644 file.txt       # rw-r--r--
chmod 600 file.txt       # rw-------
chmod 777 file.txt       # rwxrwxrwx (tidak direkomendasikan)
chmod 000 file.txt       # ----------
chmod -R 755 directory/  # Rekursif

# MODE SIMBOLIK
chmod u+x file.txt       # Tambah execute untuk owner
chmod g+w file.txt       # Tambah write untuk group
chmod o-r file.txt       # Hapus read untuk others
chmod a+x file.txt       # Tambah execute untuk semua
chmod u=rw,g=r,o= file.txt  # Set permission eksplisit
chmod go-rwx file.txt    # Hapus semua dari group dan others
chmod u+rw,g-wx file.txt # Kombinasi

# PERMISSION UMUM
chmod 755 script.sh      # Script yang bisa dieksekusi
chmod 644 config.txt     # File konfigurasi
chmod 600 private.key    # File private/secret
chmod 777 public/        # Semua bisa akses (hati-hati)
chmod 700 private/       # Hanya owner yang bisa akses
chmod 1777 /tmp          # Sticky bit + semua akses
```

### 9.4 Special Permissions

```bash
# SETUID (s) — Execute dengan privilege owner
chmod u+s /usr/bin/passwd    # Setuid
chmod 4755 /usr/bin/passwd   # Numeric

# Contoh: /usr/bin/passwd memiliki setuid
# sehingga user biasa bisa mengubah password
# meskipun harus akses /etc/shadow (milik root)

# -rwsr-xr-x  → s menggantikan x di posisi owner

# SETGID (s) — Execute dengan privilege group
chmod g+s directory/         # Setgid pada direktori
chmod 2755 directory/        # Numeric

# File baru di direktori setgid akan mewarisi group direktori
# Berguna untuk direktori yang digunakan bersama

# drwxr-sr-x  → s di posisi group

# STICKY BIT (t) — Hanya owner yang bisa hapus
chmod +t /shared/directory   # Sticky bit
chmod 1755 /shared/          # Numeric

# Di direktori sticky bit, pengguna hanya bisa hapus
# file yang mereka miliki sendiri
# Digunakan pada /tmp

# drwxrwxrwt  → t di posisi others

# Contoh:
ls -la /tmp
# drwxrwxrwt 18 root root 4096 Jan 15 10:30 /tmp
```

### 9.5 Mengubah Ownership (chown & chgrp)

```bash

chown username:group file.txt        # Ganti owner dan group
```

## 10. MANAJEMEN PROSES

### 10.1 Konsep Proses

```bash
Proses adalah program yang sedang berjalan.

Setiap proses memiliki:
  PID  (Process ID)        → Identifikasi unik
  PPID (Parent Process ID) → ID proses parent
  UID  (User ID)           → User yang menjalankan
  Priority                 → Prioritas penjadwalan
  State                    → Status proses

Status Proses:
  R  Running       → Sedang berjalan atau siap berjalan
  S  Sleeping      → Menunggu event (interruptible)
  D  Disk sleep    → Menunggu I/O (uninterruptible)
  Z  Zombie        → Selesai tapi belum diklaim parent
  T  Stopped       → Dihentikan sementara
  I  Idle          → Idle kernel thread

Hierarki Proses:
  systemd (PID 1)
  └── sshd
      └── bash
          └── vim

Hubungan:
  Parent process → Proses yang membuat proses lain
  Child process  → Proses yang dibuat oleh parent
  Orphan         → Child yang parent-nya mati
  Zombie         → Child selesai tapi belum di-wait
```

#### Script Contoh Lengkap

```bash
#!/bin/bash
# ======================================
# system_monitor.sh
# Monitor sistem dan kirim laporan
# ======================================

set -euo pipefail

# Konfigurasi
REPORT_FILE="/tmp/system_report_$(date +%Y%m%d).txt"
THRESHOLD_CPU=80
THRESHOLD_MEM=85
THRESHOLD_DISK=90

# Warna untuk output
RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
NC='\033[0m' # No Color

# Logging
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$REPORT_FILE"
}

warn() {
    echo -e "${YELLOW}[WARN]${NC} $1" | tee -a "$REPORT_FILE"
}

alert() {
    echo -e "${RED}[ALERT]${NC} $1" | tee -a "$REPORT_FILE"
}

ok() {
    echo -e "${GREEN}[OK]${NC} $1" | tee -a "$REPORT_FILE"
}

# Header
print_header() {
    echo "================================" | tee -a "$REPORT_FILE"
    echo "  System Monitor Report"          | tee -a "$REPORT_FILE"
    echo "  $(date)"                        | tee -a "$REPORT_FILE"
    echo "  Host: $(hostname)"              | tee -a "$REPORT_FILE"
    echo "================================" | tee -a "$REPORT_FILE"
}

# Check CPU
check_cpu() {
    log "Checking CPU usage..."
    local cpu_usage
    cpu_usage=$(top -bn1 | grep "Cpu(s)" | awk '{print $2}' | cut -d. -f1)
    
    if [ "$cpu_usage" -gt "$THRESHOLD_CPU" ]; then
        alert "CPU usage: ${cpu_usage}% (threshold: ${THRESHOLD_CPU}%)"
    else
        ok "CPU usage: ${cpu_usage}%"
    fi
}

# Check Memory
check_memory() {
    log "Checking memory usage..."
    local mem_total mem_used mem_percent
    mem_total=$(free -m | awk 'NR==2{print $2}')
    mem_used=$(free -m | awk 'NR==2{print $3}')
    mem_percent=$(( mem_used * 100 / mem_total ))
    
    if [ "$mem_percent" -gt "$THRESHOLD_MEM" ]; then
        alert "Memory usage: ${mem_percent}% (${mem_used}MB/${mem_total}MB)"
    else
        ok "Memory usage: ${mem_percent}% (${mem_used}MB/${mem_total}MB)"
    fi
}

# Check Disk
check_disk() {
    log "Checking disk usage..."
    while IFS= read -r line; do
        local usage mount
        usage=$(echo "$line" | awk '{print $5}' | tr -d '%')
        mount=$(echo "$line" | awk '{print $6}')
        
        if [ "$usage" -gt "$THRESHOLD_DISK" ]; then
            alert "Disk $mount: ${usage}%"
        else
            ok "Disk $mount: ${usage}%"
        fi
    done < <(df -h | grep -vE 'Filesystem|tmpfs|udev|loop')
}

# Check Services
check_services() {
    log "Checking critical services..."
    local services=("ssh" "nginx" "mysql" "cron")
    
    for service in "${services[@]}"; do
        if systemctl is-active --quiet "$service" 2>/dev/null; then
            ok "Service $service: running"
        else
            alert "Service $service: NOT running"
        fi
    done
}

# Main
main() {
    print_header
    echo ""
    check_cpu
    echo ""
    check_memory
    echo ""
    check_disk
    echo ""
    check_services
    echo ""
    log "Report saved to: $REPORT_FILE"
}

main "$@"
```

### 16.5 Cron Jobs

```bash

crontab -e                    # Edit crontab user

sudo crontab -e -u username   # Edit crontab user lain
sudo crontab -l -u username   # List crontab user lain

# Format crontab:
# * * * * * command
# │ │ │ │ │
# │ │ │ │ └── Day of week (0-7, 0&7=Sunday)
# │ │ │ └──── Month (1-12)
# │ │ └────── Day of month (1-31)
# │ └──────── Hour (0-23)
# └────────── Minute (0-59)

# Contoh:
# Setiap menit
* * * * * /path/to/script.sh

# Setiap jam
0 * * * * /path/to/script.sh

# Setiap hari jam 2 pagi
0 2 * * * /path/to/backup.sh

# Setiap Senin jam 8 pagi
0 8 * * 1 /path/to/weekly.sh

# Setiap 5 menit
*/5 * * * * /path/to/check.sh

# Hari kerja jam 9 pagi
0 9 * * 1-5 /path/to/workday.sh

# Tanggal 1 setiap bulan jam midnight
0 0 1 * * /path/to/monthly.sh

# Special strings:
# @reboot    → Saat boot
# @yearly    → Setahun sekali (0 0 1 1 *)
# @monthly   → Sebulan sekali (0 0 1 * *)
# @weekly    → Seminggu sekali (0 0 * * 0)
# @daily     → Sehari sekali  (0 0 * * *)
# @hourly    → Sejam sekali   (0 * * * *)

# CRON SYSTEM DIRECTORIES
ls /etc/cron.d/               # Job-spesifik cron files
ls /etc/cron.daily/           # Script yang jalan harian
ls /etc/cron.weekly/          # Script yang jalan mingguan
ls /etc/cron.monthly/         # Script yang jalan bulanan
ls /etc/cron.hourly/          # Script yang jalan per jam

# AT — Job sekali jalan
at 15:00                      # Jadwalkan untuk jam 3 sore
at> command
at> Ctrl+D                    # Selesai input

at now + 1 hour               # 1 jam dari sekarang
at midnight                   # Tengah malam
atq                           # Antrian at jobs
atrm 5                        # Hapus job nomor 5
```

## 17. KEAMANAN LINUX

### 17.1 Prinsip Keamanan

```
Prinsip utama keamanan Linux:
1. Principle of Least Privilege  → Berikan hak minimum yang diperlukan
2. Defense in Depth              → Berlapis-lapis pertahanan
3. Fail Secure                   → Gagal dengan aman
4. Separation of Duties          → Pisahkan tanggung jawab
5. Keep it Simple                → Complexity = vulnerabilities
6. Audit & Monitor               → Pantau dan catat semua aktivitas
```
